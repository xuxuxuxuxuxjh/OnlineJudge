  - 旧前端包未更新导致缺少 AI Modify
      - 问题：服务器上仍是历史版本的 frontend/dist，没有最新的 AI Modify 功能。
      - 处理：在本地 npm run build 生成 dist，然后 scp 上传至服务器 /root/OnlineJudge/frontend/dist 并 docker compose restart oj-backend。                                                                                                                                                                
  - 部署后 403 Forbidden（Nginx 指向空目录）                                                                                                                                                                                                                                                              
      - 问题：容器将 /app/dist 映射到宿主的 frontend/dist，但该目录为空，Nginx 无文件可返回。
      - 处理：确认 dist 中含 index.html/static/ 后再次上传并重启容器，Nginx 即返回 200。                                                                                                                                                                                                                  
  - 前端卡 Loading：vendor_b756… is not defined                                                                                                                                                                                                                                                           
      - 问题：只重新 build 了应用 JS，没有重新 build DLL，导致 oj.*.js 仍引用旧 DLL 变量名，而部署的 DLL 已换哈希。                                                                                                                                                                                       
      - 处理：在服务器 frontend/ 内修改 config/dev.env.js，增加 git 缺失时的兜底，随后 npm install、npm run build:dll、npm run build，最后 docker compose restart oj-backend。DLL 与索引文件同步后，页面加载正常。                                                                                        
  - 在服务器构建报 git rev-parse HEAD 错误                                                                                                                                                                                                                                                                
      - 原因：部署目录不是 Git 仓库。                                                                                                                                                                                                                                                                     
      - 解决：在 config/dev.env.js 里对 git rev-parse HEAD 加 try/catch，失败时用 'local' 作为版本号。                                                                                                                                                                                                    
  - npm install 报 EACCES                                                                                                                                                                                                                                                                                 
      - 原因：/root/.npm 缓存目录里有 root 所有的旧文件。                                                                                                                                                                                                                                                 
      - 解决：先 chown -R $(whoami) ~/.npm（或清空 ~/.npm），再 NODE_OPTIONS=--openssl-legacy-provider npm install。
